搭建博客过程中收集的一些方法

发表于2021-03-20|更新于2021-03-21

在使用Hexo框架搭建博客的过程中，难免会遇到各种各样的问题，本文章收集了在搭建过程中遇到问题的解决方法，希望可以帮助到其他有同样需求或者问题的人。

如何上传文件夹到Github

注：此文章转载自jojowei：如何上传文件夹到github?

第一步：新建一个仓库(Repository)

进入Github首页

点击New repository

创建一个仓库，点击左栏右上的绿色按钮New来新建一个仓库

填写信息

Repository name：仓库名称

Description(可选)：仓库描述介绍，不是必填项。

Public，Private：仓库权限。Public指公开共享，是免费的。Private指私有或指定合作者，需付费使用。

Initialize this repository with a README：添加一个README.md

gitignore：不需要进行版本管理的仓库类型，对呀生成文件.gitignore

点击Create repository

第二步：Clone or dowload仓库地址

来到新建好的repository，点击右侧中部的Code会出现一个地址，copy这个地址备用。

（这里我的地址是https://github.com/VBZE/Blog-Backup.git）

第三步：上传文件夹至Github

以下为本地操作。

新建空文件夹

本地电脑新建空文件夹。

克隆新仓库

进入文件夹，右键会出现两个选项：Git Gui Here，Git Bash Her。选择GIt Bash Here，然后把GitHub上面新建的仓库克隆到本地。

克隆仓库，代码如下：git clone + 你仓库地址

（这里我的仓库地址是https://github.com/VBZE/Blog-Backup）

上传前准备

上传步骤完成之后，本地新建的文件夹下面就会多出个文件夹。该文件夹名即为你Github上面新建的那个Repository的名字。我们把要上传的文件夹里面的内容全部复制到这个本地的Repository同名的文件夹下，然后cd进入这个克隆的文件夹：cd + 文件夹名字

我这里克隆的文件夹名为Blog-backup

我要上传的文件夹是MyBlog，我要把上传的文件夹放入Blog-Backup。然后在git界面进入Blog-Backup：cd Blog-Backup

依次输入以下代码：

git init

git add .

git commit -m "你的提交信息"

git push

依次输入以上代码后，进入GitHub查看：文件夹已上传GitHub，上传成功。

（本例中我的仓库是Blog-Backup，我要上传的文件夹是MyBlog）

【注意】：

  • git init指git初始化
  • git add .指提交到缓存区。这里的add和.中间有个空格。如果不写空格，git无法识别。add .这里就是添加所有文件的意思。
  • git commit -m "你的提交信息"是将暂存区里的改动给提交到本地的版本库。这里”你的提交信息”是你自己想要提交的信息，比如”upload”,”change setting”.
  • git push是将本地版本库的分支推送到远程服务器上对应的分支，提交到远程的GitHub仓库。
